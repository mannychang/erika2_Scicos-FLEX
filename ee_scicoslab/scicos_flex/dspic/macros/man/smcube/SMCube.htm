SMCube - palette: SMCube

Description

palette: SMCube

This block represents an instance of a state machine designed with SMCube (see references for details about SMCube usage).

Parameters

Dialog Box:

Click the Editor button to open the SMCube editor (see references for details about SMCube usage).

Click the Properties button to open/set the block preferences.

  • Number of input event ports

    This integer controls the number of activation (event) ports.

    "0" : the state machine inherit the activation from the input(s); each time one of the input is updated the state machine is evaluated.

    "1" to "32" : the state machine is explicitly activated by an external (synchronous, asynchronous or combination of the two types) event.

  • Number of output event ports

    This integer controls the length of the output event register (from "0" to "32").

  • Mode (1:interactive, 2:background)

    This parameter control the execution mode of the state machine at simulation time.

    If "1" is selected the SMCube simulation dialog will be opened to control the execution of the simulation;

    if "2" is selected insteed the SMCube will run in background.

  • Input Description ('d':double, 'i':int32)

    This parameter specify the number and type of the inputs using a synthetic notation.

    "ddd" three double inputs;

    "id" two inputs, one integer, one double, etc...

  • Output Description ('d':double, 'i':int32)

    This parameter specify the number and type of the outputs using the same synthetic notation of the inputs.

  • SMCube File

    The state machine is stored in an external XML file; you can specify just the file name (e.g. "example.xml") or the file name with path information (e.g. "C:\SMCube\Demo\example.xml")
